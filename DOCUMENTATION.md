## 1. Project Overview

A personal finance dashboard built as a Zorvyn FinTech internship assignment. It lets users track income and expenses, visualize spending trends, and switch between an Admin role (full CRUD) and a Viewer role (read-only).

**Entry point:** `src/main.tsx` renders `<App />` into `#root`.

## 4. Data Layer

### `src/types/index.ts`
Defines all shared TypeScript types used across the app:

```typescript
interface Transaction {
  id: string;          // UUID generated by crypto.randomUUID()
  date: string;        // ISO date string (YYYY-MM-DD)
  amount: number;      // Always positive; type field determines income/expense
  category: string;    // One of the 8 CATEGORIES
  type: 'income' | 'expense';
  description: string;
  createdAt: string;   // Full ISO timestamp, auto-set on creation
}

type UserRole = 'admin' | 'viewer';

interface FilterState {
  category: string;
  type: 'all' | 'income' | 'expense';
  searchQuery: string;
  sortBy: 'date' | 'amount';
  sortOrder: 'asc' | 'desc';
}

interface FinancialSummary {
  totalBalance: number;
  totalIncome: number;
  totalExpenses: number;
  monthlyIncome: number;
  monthlyExpenses: number;
}
```

### `src/lib/mockData.ts`
- Exports `CATEGORIES`: 8 strings — Salary, Food, Transport, Shopping, Entertainment, Bills, Freelance, Health.
- Exports `mockTransactions`: 18 hard-coded transactions spanning Nov 2025 – Apr 2026.
- Used as the initial state in `FinanceContext` when `localStorage` is empty (first visit).

## 5. State Management

The app uses React's built-in **Context API** — no Redux, no Zustand.

### `src/context/FinanceContext.tsx`

**What it holds:**
- `transactions: Transaction[]` — the master list, initialized from localStorage or mock data.
- `role: UserRole` — current user role, initialized from localStorage.
- `summary: FinancialSummary` — computed with `useMemo` from `transactions` on every change. Calculates total balance, income, expenses, and this-month figures.

**CRUD functions exposed:**
- `addTransaction(t)` — prepends a new transaction with a UUID and ISO timestamp.
- `updateTransaction(id, updates)` — maps over array, replaces matching item.
- `deleteTransaction(id)` — filters out the matching item.
- `setRole(role)` — switches between admin and viewer.

**localStorage sync:** Two `useEffect` hooks watch `transactions` and `role` and write to `localStorage` on every change.

**How to use in a component:**
```typescript
const { transactions, role, addTransaction, summary } = useFinance();
```

### `src/context/ThemeContext.tsx`

**What it holds:**
- `theme: 'light' | 'dark'` — initialized from localStorage (defaults to `'light'`).
- `toggleTheme()` — flips the theme.

**DOM sync:** A `useEffect` adds/removes the `.dark` class on `document.documentElement` whenever `theme` changes. Tailwind v4 reads this class to apply dark-mode utilities.

**How to use:**
```typescript
const { theme, toggleTheme } = useTheme();
```

## 6. Layout & Navigation

### `src/App.tsx`
- Wraps everything in `<ThemeProvider>` then `<FinanceProvider>`.
- Inner `AppContent` component holds all page-level state:
  - `activeTab` — which tab is currently shown (`'dashboard'`, `'transactions'`, `'insights'`).
  - `filters` — the current `FilterState` object for the transactions tab.
  - `modalOpen` / `editingTransaction` — modal visibility and which transaction to edit.
- `filteredTransactions` is derived with `useMemo` from `transactions + filters` — no extra state.
- Each tab section is wrapped in `<div key="..." className="tab-content">` — the `key` prop forces React to remount the div on tab switch, which re-triggers the CSS animation.

### `src/components/layout/MainLayout.tsx`
- Holds `sidebarOpen` boolean state.
- Renders `<Header>`, `<Sidebar>`, and a `<main>` content area side-by-side.
- Passes `onMenuToggle` to Header (opens sidebar) and `isOpen/onClose` to Sidebar.

### `src/components/layout/Header.tsx`
The top bar with three sections:

1. **Hamburger button** — visible on mobile only (`lg:hidden`), calls `onMenuToggle`.
2. **Animated role dropdown** — custom-built (not a `<select>`):
   - `useState` tracks open/closed.
   - `useRef` + `useEffect` detects clicks outside to close it.
   - The dropdown panel uses CSS `opacity`, `scale-y`, and `translate-y` with a 200ms transition for smooth open/close.
   - Active role gets a blue dot indicator and blue background highlight.
   - `ChevronDown` icon rotates 180° when open via `rotate-180` class.
3. **Theme toggle button** — shows Moon icon in light mode, Sun icon in dark mode.

### `src/components/layout/Sidebar.tsx`
- Tab list: Dashboard, Transactions, Insights — each a button with an icon from Lucide.
- **Desktop** (`lg:`): always visible as a static sidebar.
- **Mobile**: fixed positioned, slides in from the left via `translate-x` transition. A semi-transparent black backdrop appears behind it and closes it when clicked.
- Active tab gets `bg-blue-50 text-blue-600` (light) / `bg-blue-900/30 text-blue-400` (dark) highlight.

## 7. Dashboard Tab

### `src/components/dashboard/SummaryCards.tsx`
Renders 3 cards: Total Balance, Total Income, Total Expenses.

- Each card uses the `SummaryCard` sub-component.
- Each calls `useCountUp(Math.abs(amount), 1000)` — the displayed number animates from 0 to the real value over 1 second on mount.
- Negative balance shows a `-` prefix and red color.
- Icons: `DollarSign`, `TrendingUp`, `TrendingDown` from Lucide.

### `src/components/dashboard/BalanceTrendChart.tsx`
A 6-month running balance area chart.

**Data computation (`useMemo`):**
- Builds an array of the last 6 calendar months.
- For each month, sums all income (positive) and expenses (negative) from `transactions`.
- Accumulates a `runningBalance` that carries over between months.

**Chart details (Recharts `AreaChart`):**
- SVG `<defs>` inside the chart defines two gradients:
  - `lineGradient` — horizontal, indigo → blue → cyan, used as the line stroke color.
  - `areaGradient` — vertical, blue-30% → transparent, used as the fill below the line.
- `<Area>` component: `stroke="url(#lineGradient)"`, `fill="url(#areaGradient)"`, strokeWidth=3.
- Animated: `animationDuration={1200}`, `animationEasing="ease-out"` — the line grows from left to right on first render.
- Dots at each data point with white border.
- Custom `<ChartTooltip />` for hover popup.

### `src/components/dashboard/SpendingBreakdownChart.tsx`
A pie chart showing expense totals per category.

**Data computation (`useMemo`):**
- Filters transactions to expenses only.
- Groups by `category`, sums amounts.
- Returns `[{ name, value }]` array for Recharts.

**Chart details (Recharts `PieChart`):**
- SVG `<defs>` defines 8 radial gradients (one per possible category color) with `id="pie-gradient-0"` through `id="pie-gradient-7"`.
- Each `<Cell>` uses `fill="url(#pie-gradient-N)"` for the gradient slice fill.
- Labels show percentage via `label={({ percent }) => ...}`.
- `<Legend iconType="circle" iconSize={10} />` — round color dots in the legend (not squares).
- Animated on mount with `animationDuration={1000}`.
- Custom `<ChartTooltip />` for hover popup.

## 8. Transactions Tab

### `src/components/transactions/TransactionFilters.tsx`
Four filter controls rendered in a flex-wrap row:

1. **Search input** — filters by description or category (case-insensitive `.includes()`).
2. **Category dropdown** — select from all 8 categories, or "All Categories".
3. **Type dropdown** — All / Income / Expense.
4. **Sort dropdown** — Date Newest, Date Oldest, Amount High, Amount Low.

All four are controlled inputs that call `onChange(updatedFilters)` on every change. The parent (`App.tsx`) holds the `filters` state and passes down `filteredTransactions` to the table.

### `src/components/transactions/TransactionTable.tsx`
A standard HTML `<table>` with columns: Date, Description, Category, Type, Amount, Actions.

- **Category badge:** gray pill (`rounded-full`).
- **Type badge:** green pill for income, red pill for expense.
- **Amount:** green `+$` for income, red `-$` for expense.
- **Row hover animation:** `hover:scale-[1.01] hover:shadow-sm transition-all duration-150` — rows slightly grow on hover.
- **Actions column** (admin only): Edit button (pencil icon) and Delete button (trash icon). Delete calls `deleteTransaction(id)` directly. Edit calls `onEdit(transaction)` which opens the modal in App.tsx.
- Empty state: shows a "No transactions found" message.

### `src/components/transactions/TransactionModal.tsx`
A modal overlay for adding or editing a transaction.

- **Backdrop:** fixed full-screen dark overlay, clicks on it close the modal.
- **Form fields:** Description, Amount, Date, Category (dropdown), Type (dropdown).
- **Controlled form state:** single `form` object with `useState`, reset by a `useEffect` that runs whenever `isOpen` or `transaction` prop changes.
- **Validation:** all fields required; amount must be a positive number. Shows a red error box if invalid.
- **Submit logic:** if `transaction` prop is set → calls `updateTransaction()`, else → calls `addTransaction()`.
- Title changes between "Add Transaction" and "Edit Transaction" based on whether a transaction is passed.

## 9. Insights Tab

### `src/components/insights/InsightsSection.tsx`

**Three stat cards** (all with count-up animations and colored left borders):

1. **Top Spending** (yellow left border) — finds the expense category with the highest total. Shows category name and total amount animated from 0.
2. **This Month** (red left border) — shows current month's total expenses vs. previous month's. Both numbers are animated.
3. **Month-over-Month** (green if spending decreased, red if increased) — shows the percentage change between this month and last month, animated. Icon and color flip based on whether spending went up or down.

**Spending by Category Bar Chart (Recharts `BarChart`):**
- `<defs>` with 8 vertical `<linearGradient>` elements (one per bar color) — each gradient goes from full opacity at top to 50% opacity at bottom.
- Each `<Bar>` uses `<Cell fill="url(#gradient-N)">` for per-bar gradient coloring.
- `radius={[6, 6, 0, 0]}` — rounded top corners on bars.
- Animated with `animationDuration={800}`.
- Custom `<ChartTooltip />` for hover popup.

**Category Breakdown (progress bars):**
- One row per category, sorted by amount descending.
- Each row: category name on left, dollar amount on right.
- Progress bar: a gray track (`bg-gray-200`) with a colored fill div whose `width` is set via inline style `(amount / maxAmount) * 100%`.
- Fill color set via inline `backgroundColor: BAR_COLORS[index]` — same palette as the bar chart.
- `transition-all duration-700 ease-out` — bars animate their width on mount.

## 10. Shared Utilities

### `src/lib/useCountUp.ts`
Custom React hook that animates a number from 0 to a target value.

```
useCountUp(target: number, duration = 1000) → number
```

**How it works:**
1. Stores the current displayed count in `useState(0)`.
2. On `target` change, records `startTime = performance.now()`.
3. Uses `requestAnimationFrame` to call a `tick` function on each frame.
4. Each frame: calculates `progress = elapsed / duration` (clamped to 1).
5. Applies cubic ease-out: `eased = 1 - (1 - progress)^3` — starts fast, decelerates at the end.
6. Sets `count = Math.round(eased * target)`.
7. Stops when `progress >= 1`.

Used in: `SummaryCards.tsx` (3 cards) and `InsightsSection.tsx` (4 values).

### `src/lib/utils.ts`
Single function `cn(...inputs)` — combines `clsx` (conditional classes) and `tailwind-merge` (deduplicates conflicting Tailwind classes). Used throughout for readable conditional className strings.

### `src/components/ui/ChartTooltip.tsx`
Custom tooltip component used by all three Recharts charts (`BalanceTrendChart`, `SpendingBreakdownChart`, `InsightsSection`).

**Why it exists:** Recharts' built-in `formatter` prop has type incompatibilities in newer versions that cause empty tooltip containers. This component bypasses that entirely by receiving the `payload` array directly.

**Props:**
- `active` — whether the tooltip is currently visible.
- `payload` — array of `{ name, value, color }` from Recharts.
- `label` — the x-axis label (e.g., month name).
- `prefix` — prepended to value (default `'$'`).
- `suffix` — appended to value (default `''`).

**Renders:** a white/dark-gray card with a colored circle dot, data series name, and formatted value for each entry.

## 12. Dark / Light Mode

**The problem with Tailwind v4:** By default, Tailwind v4 uses `prefers-color-scheme` media query for dark mode. This means dark classes would always be active on systems set to dark OS theme, ignoring the toggle.

**The fix in `src/index.css`:**
```css
@custom-variant dark (&:where(.dark, .dark *));
```
This tells Tailwind to only apply `dark:` utilities when a `.dark` class exists on a parent element.

**ThemeContext adds/removes `.dark` on `<html>`:**
```typescript
document.documentElement.classList.add('dark');    // dark mode
document.documentElement.classList.remove('dark'); // light mode
```

**Color scheme used:**
- Light: `bg-slate-100` page, `bg-slate-50` cards, `text-gray-900`.
- Dark: `bg-gray-900` page, `bg-gray-800` cards, `text-gray-50`.

## 13. Role-Based Access Control

Two roles: `admin` and `viewer`.

| Feature | Admin | Viewer |
|---|---|---|
| View all data | Yes | Yes |
| Add Transaction button | Visible | Hidden |
| Edit button per row | Visible | Hidden |
| Delete button per row | Visible | Hidden |
| Actions column in table | Shown | Hidden |
| Role switching | Available to both | Available to both |

Role is checked directly from `useFinance()`:
```typescript
const { role } = useFinance();
{role === 'admin' && <button>Add Transaction</button>}
```

Role is persisted in `localStorage` so it survives page refresh.

## 14. localStorage Persistence

Three keys are stored:

| Key | Value | Where set |
|---|---|---|
| `finance_transactions` | JSON array of all transactions | `FinanceContext` via `useEffect` |
| `finance_role` | `'admin'` or `'viewer'` | `FinanceContext` via `useEffect` |
| `finance_theme` | `'light'` or `'dark'` | `ThemeContext` via `useEffect` |

**Initial load pattern (lazy initializer):**
```typescript
const [transactions, setTransactions] = useState<Transaction[]>(() => {
  const saved = localStorage.getItem('finance_transactions');
  return saved ? JSON.parse(saved) : mockTransactions;
});
```
The function inside `useState(...)` only runs once on mount, not on every render.

## 15. How Everything Connects

```
index.html
  └── main.tsx
        └── App
              ├── ThemeProvider  (wraps everything)
              │     └── ThemeContext — theme, toggleTheme
              │
              └── FinanceProvider  (wraps AppContent)
                    └── FinanceContext — transactions, role, summary, CRUD
                          │
                          └── AppContent
                                ├── activeTab state  ──────────────────────────┐
                                ├── filters state (FilterState)                │
                                ├── modalOpen / editingTransaction state        │
                                ├── filteredTransactions (useMemo)              │
                                │                                               │
                                └── MainLayout ─── passes activeTab / onTabChange
                                      ├── Header
                                      │     ├── reads: theme, toggleTheme (ThemeContext)
                                      │     └── reads: role, setRole (FinanceContext)
                                      │
                                      ├── Sidebar  ← receives activeTab, onTabChange ──┘
                                      │
                                      └── <main> children (conditional on activeTab)
                                            ├── Dashboard tab
                                            │     ├── SummaryCards  ← reads summary
                                            │     ├── BalanceTrendChart  ← reads transactions
                                            │     └── SpendingBreakdownChart  ← reads transactions
                                            │
                                            ├── Transactions tab
                                            │     ├── TransactionFilters  ← reads categories
                                            │     ├── TransactionTable  ← reads role, deleteTransaction
                                            │     └── TransactionModal  ← reads addTransaction, updateTransaction
                                            │
                                            └── Insights tab
                                                  └── InsightsSection  ← reads transactions
```

**Data always flows one way:** Context → Components → UI. Components never modify context directly — they call the functions exposed by the context (`addTransaction`, `deleteTransaction`, etc.).
